# LwM2M Client Conformance & Interoperability Tool

## Conformance Validation Based on Official OMA Interoperability Test Specifications

The **OMA LwM2M Client Conformance Tool** enables device manufacturers, software developers, and interoperability engineers to independently verify that their LwM2M client implementation conforms to OMA specifications before deployment, interoperability events, or customer trials.

The tool executes an automated suite of conformance and interoperability tests against an **OMA-hosted Eclipse Leshan reference server**, validating client behaviour across **LwM2M 1.0** and **LwM2M 1.1** implementations.

Tests are derived directly from official OMA normative specifications, including:

* [**LwM2M Interoperability Test Specification**](https://www.openmobilealliance.org/release/LightweightM2M/ETS/)
* [**Smart City Conformance Test Specification**]()

Supported targets include real devices, software implementations, gateways, and bridge or connector solutions that interconnect legacy or non-IP devices.

At the completion of each test run, the tool generates a detailed HTML report identifying `passed`, `failed`, and `incomplete` test cases, helping engineering teams rapidly identify implementation issues and accelerate interoperability readiness.

## Why Use the Conformance Tool?

### Validate Against Official OMA Test Cases

Execute tests derived directly from OMA normative specifications rather than proprietary or vendor-specific test environments.

### Reduce Interoperability Risk

Identify protocol implementation issues before participating in interoperability events, field trials, or production deployments.

### Accelerate Debugging

Receive detailed pass/fail reports that help pinpoint exactly which client behaviours require attention.

### Demonstrate Standards Compliance

Generate objective conformance evidence that can support procurement requirements, customer evaluations, and RFP responses.

## What It Tests

### Real Devices and Software Implementations

Validate:

* Embedded LwM2M clients
* Smart meters and utility devices
* IoT sensors and actuators
* Gateways and edge platforms
* Bridge and connector solutions
* Software-only client implementations

Testing can be performed across standard IP networks including:

* NB-IoT
* LTE-M
* Wi-SUN
* 6LoWPAN
* Ethernet
* Wi-Fi

### Core LwM2M Client Operations

The tool validates behaviour across fundamental LwM2M interfaces and procedures, including:

* Registration and Registration Update
* Bootstrap procedures
* Read
* Write
* Execute
* Create
* Delete
* Observe
* Notify

Additional coverage includes:

* LwM2M 1.1 enhancements
* Composite Operations
* Information Reporting
* OSCORE Security

### Profile-Based Test Execution

The tool supports multiple test profiles covering both interoperability and Smart City use cases, allowing organizations to execute tests that are relevant to their implementation and deployment scenario.

### Detailed HTML Reports

Each execution produces a structured HTML report containing:

* Test case results
* Pass/fail status
* Execution details
* Diagnostic information

This provides a permanent record of validation activities and enables rapid troubleshooting of implementation issues.

## Architecture

The conformance tool operates as an automated test harness that validates a Device Under Test (DUT) through an OMA LwM2M-hosted Eclipse Leshan reference server.

The DUT communicates with the server over a standard IP network while the conformance tool executes test procedures and verifies client responses against expected protocol behaviour.
